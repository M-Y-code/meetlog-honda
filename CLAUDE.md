# 商談録音・要約・ToDoアプリ（個人利用専用）

HONDA営業担当者本人のみが使う、iPhone用の録音・要約・ToDo管理アプリ。
配布・複数ユーザー対応・社内システム連携は行わない。すべての処理を端末内で完結させる。

## 開発の進捗状況（最終更新: 2026-07-20）

- **リポジトリ**: https://github.com/M-Y-code/meetlog-honda （パブリック、`main`ブランチ）
- **現在のフェーズ**: Phase 1（PoC）のコード一式を実装済み。**GitHub Actions上でのシミュレータビルドが成功する状態**（commit `32b65b8` 時点）
- **開発環境**: 作業者はWindows機で、Macを持っていない・購入もしない方針。そのため以下の無料構成を採用：
  - コードはWindows上で編集し、`git push`
  - ビルド確認は**GitHub Actionsのパブリックリポジトリ無料枠のmacOSランナー**で実施（`.github/workflows/build.yml`）
  - 実機インストールは**AltStore/AltServer**（無料Apple IDでのサイドロード、AltServerはWindows上で動作）を使う予定。Apple Developer Program（年間$99）は使わない
- **実装済みファイル一式**: `project.yml`（XcodeGen定義）、`Sources/MeetLog/` 以下に4タブのSwiftUI画面・録音/文字起こし/要約サービス・ToDo/履歴ストア一式（詳細は[README.md](./README.md)の「現時点の実装範囲」を参照）

### ここまでで踏んだ落とし穴（同じ問題を繰り返さないためのメモ）
- `FoundationModels` フレームワークは **Xcode 26 / iOS 26 SDK以降でないと存在しない**。GitHub ActionsのmacOSランナーはデフォルトで古いXcodeを向いていることがあるため、CIでは `ls -d /Applications/Xcode_*.app | sort -V | tail -n 1` で明示的に最新版を選択している
- `project.yml` の `deploymentTarget` / `IPHONEOS_DEPLOYMENT_TARGET` は `26.0` にしてある（FoundationModelsの最低要件に合わせるため）
- `xcodebuild` の生ログは新ビルドシステムがJSON形式（`-use-frontend-parseable-output`）で出力するため、`grep "error:"` では実際のコンパイルエラーを拾えない。CIでは **`xcbeautify`** をパイプして読みやすい形に整形している
- `FoundationModels` だけをimportしても `Date` 等のFoundation型は使えない（`import Foundation` を別途書く必要がある）
- SwiftUIの三項演算子で `.secondary`（HierarchicalShapeStyle）と `.red`（Color）を混ぜると型エラーになる。両辺を `Color.xxx` で揃える必要がある

### 次にやること（Step 2〜3、順番に進める方針で合意済み）
1. **Step 1（進行中）**: AltServerのセットアップ（iTunes公式版インストール → AltServerインストール → iPhoneをUSB接続・信頼 → AltServerから「Install AltStore」→ iPhone側で「設定 > 一般 > VPNとデバイス管理」でデベロッパープロファイルを信頼）。ユーザーが手元で作業中、完了報告待ち

## 確定仕様サマリー

- **利用形態**: 自分専用。配布・複数ユーザー対応・社内展開は無し。App Store審査も通さない
- **処理方式**: 録音・文字起こし・要約・ToDo抽出まで全て端末内で完結。通信不要、バックエンドなし
- **対応機種**: Apple Intelligence対応iPhone（iPhone 15 Pro以降）
- **配布方法**: Apple Developer Program（有料）は使わず、**無料構成**（GitHub Actionsのパブリックリポジトリ無料枠でビルド → AltStore/AltServerで無料Apple IDサイドロード）で自分のiPhoneへインストールする

## 主要機能

### 録音
- ワンタップ開始/停止。ロック画面・ウィジェットからも操作可能
- 数時間の連続録音に対応（バックグラウンド録音、省電力エンコード）
- カテゴリ（商談／社内会議／電話／メモ／その他）とタイトルで自由にタグ付け。商談に限らず何でも登録可能

### 並行文字起こし・リアルタイム要約
- 録音と同時にオンデバイスでストリーミング文字起こし
- 無音区間で自然に区切った「チャンク」（3〜5分目安、最大7分で強制分割）単位で**区間要約**を生成
- 区間要約を3件たまるか10〜15分経過するごとにリファインし、「**累積要約**」として全体像を常に最新化
- 商談中に「要約を見る」パネルを開けば、その時点の累積要約をいつでも確認できる（2分以上古ければ開いた時点で即リフレッシュ）
- 録音停止時に累積要約を仕上げて最終要約（概要／要望・懸念点／決定事項・保留事項）を生成

### 汎用ToDo管理・リマインド
- 商談に紐づかないタスクも自由に登録可能（買い物・社内タスク・個人予定など何でも）
- 要約からAIがToDo候補を自動抽出。**区間要約の時点で都度フラグ**しておき、終盤の要約に埋もれて欠落するのを防ぐ
- 期限・優先度・カテゴリを個別設定し、ローカル通知でリマインド

### 履歴管理
- カテゴリ・キーワードで検索可能
- 一覧に「削除まで残りN時間」のカウントダウンを常時表示

## データライフサイクル

### 録音・文字起こし・要約・同意ログ
- 作成から**48時間で例外なく完全削除**（延長操作は一切提供しない。電波不良等で未処理でも例外なし）
- 削除24時間前・46時間後（削除2時間前）に警告通知
- 削除前に限り、**最終要約テキストのみ**コピー・共有可能（音声・全文文字起こしの持ち出しは不可）
- ユーザーの任意操作による即時削除も可能

### ToDo
- 録音データとは独立したライフサイクルで管理（48時間ルールの対象外）
- **完了チェックを入れたら即座に削除**
- **期限到来（未完了）** → 期限切れ通知＋**24時間の猶予期間**（延長・完了・即削除を選択可）→ 猶予終了で自動完全削除

## 同意取得フロー

- カテゴリが「商談」の場合、録音開始前に同意確認画面を必ず表示（声かけ例文を提示）
- 「相手の同意を得ました」にチェックしない限り録音は開始できない
- 同意が得られない場合は、録音なしのテキストメモモードへ切り替え
- 同意フラグ・時刻のログは録音データと同一ライフサイクルで扱い、48時間で一緒に完全削除（監査目的の長期保持はしない）
- 商談以外のカテゴリでは、設定タブで同意確認の要否を切り替え可能

## 技術構成

| 領域 | 採用技術 |
|---|---|
| プラットフォーム | iOS（Swift / SwiftUI） |
| 音声認識 | `SFSpeechRecognizer`（`requiresOnDeviceRecognition = true`） |
| 要約・ToDo抽出 | Foundation Models framework（Apple Intelligence 内蔵オンデバイスLLM） |
| 通信 | 不要。全処理が端末内で完結 |
| バックエンド | なし |
| データ保存 | 端末内ローカルストレージのみ、暗号化して保持 |
| 通知 | ローカル通知（APNs不要） |

通信を使わない設計のため、オフライン時のキューイングや復旧後の再処理は不要。電波状況に関わらず常にその場で文字起こし・要約が完結する。

## 画面構成（4タブ）

- **録音**: カテゴリ選択 → 同意確認 → 録音中（要約パネル）→ サマリー確認
- **履歴**: 録音・要約一覧、削除カウントダウン、検索・絞込
- **ToDo**: 期限順一覧、手動追加、期限切れ・猶予表示
- **設定**: 通知タイミング、同意確認ON/OFF、録音品質、即時削除

## 実装時に検証すべき点

- Foundation Models frameworkによる要約・ToDo抽出の精度（特に日本語ビジネス文書としての実用性）を早期にプロトタイプで確認する
- 数時間の連続録音時のバッテリー消費・ストレージ使用量・端末発熱を実機で検証する
- 長時間録音中のオンデバイス文字起こしの処理負荷（区間要約生成とのリソース競合）を確認する

## 開発ロードマップ（個人開発版）

1. **PoC**: 長時間録音＋オンデバイス文字起こしの試作、Foundation Modelsによる要約品質の確認
2. **並行要約**: 区間要約・累積要約のローリング更新ロジックを実装
3. **ToDo・通知**: 汎用ToDo登録、リマインド、期限切れ猶予フローを実装
4. **データ消去**: 48時間自動削除バッチ、同意フローを実装
5. **実運用開始**: 自分の端末にインストールして日常利用を開始

社内承認プロセスや試験導入フェーズは不要なため、個人開発として柔軟なペースで進行してよい。
